Welcome to Darren and Doug's Kea and Stork Workshop

Kea is an open-source DHCPv4 and DHCP6 software system from ISC (Internet Systems Consortium). Stork is a web-based management system that makes it easier to monitor and configure a network of Kea servers. Both Kea and Stork are applications run under *NIX operating systems. ISC publishes both the source code, and pre-compiled packages for popular operating systems.

This workshop is for people who want to run DHCP servers for their network who want to get some quick exposure to the Kea DHCP system. Participants should be familiar with the Linux CLI, novice level is ok, and should have access to an ssh client.

Users will be given VMs and walked through the installation of two Kea DHCP servers and a client simulator, which will generate traffic.

  • After installation, we will configure the Kea servers, establishing one as the primary and the other as a backup system, and adding a subnet, a pool and some logging.
  • After starting everything up, we will run a few tests to determine that everything is operational, including generating some client traffic to see addresses assigned.
  • We will then install the Stork graphical management system and the Stork agent, authorize our Kea servers for monitoring and allow everyone time to explore the interface.
  • Depending on the group's progress, we will trigger a failover event, to test the high availability set-up.
  • After testing the high availability setup, we will then demonstrate and allow users time to follow along with editing the configuration.
